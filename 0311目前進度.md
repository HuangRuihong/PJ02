# Split-it-Smart 專案進度報告 (2026-03-11)

## 📋 專案概況
本專案為 **Split-it-Smart**，一個專為室友與好友設計的「非同步分帳與債務管理系統」。系統核心目標是透過非同步確認機制，解決多人記帳時的時序混亂與衝突問題。

## 🏗️ 核心架構進度
目前系統已完成基礎架構的搭建與模組化重構：

### 1. UI 層 (User Interface)
*   **模組化完成**：已將 UI 拆分為 `ui/personal/` (個人)、`ui/group/` (群組) 與 `ui/components/` (通用元件)。
*   **現代化介面**：基於 `customtkinter` 實作，支援深色模式與流暢的分頁導覽。
*   **好友系統**：已整合掃描 QR 碼名片的功能。

### 2. Core 層 (Service Logic)
*   **解耦設計**：`PersonalService` 與 `GroupService` 完全分離，確保平行開發的穩定性。
*   **資料庫事務**：全面採用 SQLite 事務保護機制，確保數據寫入的原子性。

### 3. 資料庫 Schema (`doc/schema.sql`)
*   已定義 `groups`、`transactions` 與 `transaction_participants` 表，支援顆粒度細緻的狀態追蹤（Pending -> Confirmed -> Settled）。

## ✅ 已實現功能
- [x] **群組管理**：透過 4 位隨機邀群碼建立、加入群組。
- [x] **非同步分帳**：新增交易時可選擇參與者，系統自動計算平分金額（處理餘數誤差）。
- [x] **確認機制**：參與者需手動點擊「確認」後，該帳項才正式併入總計。
- [x] **好友快速連結**：QR 碼名片生成與文件掃描功能。

## 📅 開發階段與里程碑
目前專案處於 **Phase 2 (功能實戰開發)**：
- **M1 (數據閉環)**：已完成。個人與群組數據能實時同步。
- **M2 (還款實測)**：進行中。正在完善結帳與還款的自動抵銷演算法。
- **M3 (最終校準)**：待進行。預計進行效能優化與 Bug 修復。

## 🚀 後續計畫
1.  **完善還款邏輯**：實現「一鍵結案」功能，將已確認的債務轉化為還款紀錄。
2.  **數據導出**：實作 CSV 報表匯出功能。
3.  **UI 拋光**：增加更多的微動畫與視覺反饋。
