# 專案資料夾架構說明 (Modularized Version)

本文件詳述 **Split-it-Smart (Group Ledger)** 專案模組化後的目錄結構與各組件權責。

## 核心目錄樹狀圖

```text
group ledger/ (專案根目錄)
├── run_app.py               # 應用程式啟動入口 (主程式入口)
├── main_gui.py              # 主 GUI 核心與分頁導覽邏輯
├── analysis/                # 【報表模組】數據統計與圖表視圖
│   ├── analysis_frame.py    # 圓餅圖/長條圖統計分析分頁
│   └── calendar_frame.py    # 日曆交易分布視圖
├── auth/                    # 【驗證模組】安全與權限管理
│   └── auth_ui.py           # 登入介面與自動登入邏輯
├── doc/                     # 【文件模組】開發紀錄與說明
│   ├── 資料夾架構.md          # (本文件) 當前專案結構說明
│   ├── migrations.sql       # 資料庫變更歷史 SQL
│   ├── schema.sql           # 全域資料表 DDL 定義
│   ├── 專案完整說明文件.md    # 核心功能與邏輯說明
│   └── old_docs/            # 過往版本的舊型文件存檔
├── groups/                  # 【群組模組】群組管理與多人同步
│   ├── group_frame.py       # 群組中心 UI 介面
│   └── group_service.py     # 分帳演算法與結算邏輯
├── intelligence/            # 【智慧模組】業務邏輯層
│   ├── debt_system.py       # 本地核心債務狀態機 (State Machine)
│   └── network_facade.py    # 網路通訊代理層 (API 介接)
├── personal/                # 【個人模組】個人帳務與社交
│   ├── friends_frame.py     # 好友列表與名片管理

│   ├── models.py            # 資料模型 (Data Class) 與列舉定義
│   └── data/                # 數據存放區 (Database & Config)
│       ├── accounting.db    # 本地主資料庫
│       └── server_accounting.db # 伺服器端同步資料庫
└── tools/                   # 【工具工具】營運與自動化腳本
    ├── db_update.py         # 資料庫結構自動更新腳本
    ├── sync_latest.bat      # 一鍵本地程式碼同步工具
    ├── update_db.bat        # 資料庫遷移與結構校驗工具
    ├── upload_changes.bat   # 一鍵推送到伺服器工具
    └── upload_changes.py    # 自動化 Git 上傳核心腳本
```

## 模組權責說明

### 1. 業務邏輯與智慧層 (Intelligence & Services)
- **`intelligence/`**: 存放系統的大腦。`debt_system.py` 負責處理交易狀態的轉換（PENDING -> CONFIRMED -> SETTLED）以及全域的債務計算。
- **`shared/models.py`**: 定義了整個系統通用的資料格式，確保前後端與資料庫之間的一致性。

### 2. 使用者介面 (Frontend UI)
- **`main_gui.py`**: 應用的主骨幹，負責管理側邊欄導覽與模組分頁的切換。
- **`shared/dialogs.py`**: 封裝了所有彈窗互動，保持介面代碼的簡潔。

### 3. 資料持久化 (Data Layer)
- **`shared/base_service.py`**: 所有 Service 類別的父類別，處理資料庫連線的建立與關閉。
- **`shared/data/`**: 存放所有持久化數據，包含本地資料庫檔案。

### 4. 自動化工具 (Automation Tools)
- **`tools/`**: 提供開發者與使用者便利的維護工具，包含版本控制上傳與資料庫一鍵更新。

---

## 開發規範建議
- **引用途徑**: 所有模組引用應基於根目錄出發（例：`from shared.models import ...`）。
- **資源存放**: 所有的資料庫變動必須在 `doc/migrations.sql` 同步紀錄。
