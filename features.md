# 分散式記帳系統 (Split-it-Smart) 功能清單

本系統已實作下列核心功能，旨在提供高效、精確且具備社交彈性的分帳體驗。

## 1. 核心帳本邏輯 (Core Ledger)
- **參與者層級狀態機 (Participant-level State Machine)**：
    - 每位分帳人的確認狀態獨立。一旦某人確認，該筆債務對其立即生效。
    - 支援「拒絕 (Reject)」機制，異議不影響其他成員已成立的帳目。
- **高精度金額計算**：
    - 內部全面使用「分 (Integer)」作為儲存與運算單位。
    - 徹底解決 `0.1 + 0.2` 的浮點數累計誤差問題。
- **餘數自動分配**：
    - 平均分攤時，若金額除不盡（如 10 元分三人），系統自動分配 1 分的差額，確保總和絕對正確。

## 2. 彈性分帳功能 (Flexible Splitting)
- **平均分攤 (Equal Split)**：一鍵平分總額。
- **自訂金額 (Custom Split)**：支援手動輸入每人具體負擔的金額，滿足非對稱的分帳需求。
- **即時反映餘額**：主介面餘額卡片會根據「已確認」的明細動態更新，提供即時的結算數據。

## 3. 自動化管理 (Automation)
- **金額分級催告 (Tiered Dunning)**：
    - 根據代墊金額大小自動設定還款/確認期限。
    - **小額 (< 500)**：預設 5 天期限。
    - **大額 (>= 500)**：預設 3 天期限，提升資金回籠速度。
- **系統化法制提示**：逾期交易會在系統後端標記並可供 UI 顯示警告提示。

## 4. 現代化 UI 介面 (GUI Features)
- **深色模式介面**：基於 CustomTkinter 打造，美觀且不傷眼。
- **消費熱度統計 (Expense Heatmap)**：
    - 側邊欄顯示各成員的代墊活躍度長條圖。
    - 直觀看出誰最常幫大家墊錢。
- **待確認動態清單**：
    - 區分「與我有關」與「無關人士」的帳目。
    - 視覺化狀態圖標：✅ (已確認)、❌ (已拒絕)、⏳ (等待中)。
