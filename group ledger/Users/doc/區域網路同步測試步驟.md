# 區域網路 (LAN) 資料同步測試步驟

本文件描述 **Group Ledger (分帳系統)** 在同一區域網路 (LAN) 下，兩台電腦進行即時資料同步的測試流程。

---

## 角色分工

| 角色 | 執行檔 | 說明 |
| :--- | :--- | :--- |
| **筆電 A（伺服器）** | `server\run_online_A.bat` | 啟動 API 伺服器，負責儲存與處理所有資料 |
| **筆電 B（客戶端）** | `Users\run_online_B.bat` | 連線至伺服器，以網路模式啟動 GUI |

---

## 步驟一：啟動伺服器（筆電 A）

1. 雙擊執行 `server\run_online_A.bat`。
2. 畫面會自動顯示本機 IP 位址與連線網址，例如：
   ```
   [Info] Server IP Address : 10.30.21.113
   [Info] Client should connect to : http://10.30.21.113:8000
   ```
3. 看到以下訊息代表伺服器啟動成功，**保持此視窗開啟**：
   ```
   INFO:     Uvicorn running on http://0.0.0.0:8000
   ```

> ⚠️ 若出現 `[Errno 10048]` 錯誤（Port 被占用），請在 PowerShell 執行以下指令後重試：
> ```powershell
> Get-Process python | Stop-Process -Force
> ```

---

## 步驟二：啟動客戶端（筆電 B）

1. 確保筆電 B 與筆電 A 連接在**同一個 Wi-Fi**。
2. 雙擊執行 `Users\run_online_B.bat`。
3. 當提示輸入 URL 時，輸入筆電 A 畫面所顯示的網址（例如）：
   ```
   http://10.30.21.113:8000
   ```
4. 按 Enter，程式以**聯網模式 (Network Mode)** 啟動。

> 💡 若輸入網址後出現「Mode: Local」，代表連線失敗，請參考「故障排除」章節。

---

## 步驟三：同步功能驗證

| 步驟 | 筆電 A 操作 | 筆電 B 觀察 | 預期結果 |
| :--: | :--- | :--- | :--- |
| 1 | 以 `Alice` 登入，建立新群組「測試群組」，取得邀請碼。 | 以 `Bob` 登入。 | 雙方均能正常登入。 |
| 2 | 將邀請碼告知夥伴。 | 使用邀請碼加入群組。 | 筆電 B 成功加入群組。 |
| 3 | 新增交易：Alice 支付 $500，由 Alice 和 Bob 平均分擔。 | 等待 5 秒（自動重新整理）。 | 筆電 B 交易列表出現新交易，狀態為「待確認」。 |
| 4 | 查看個人中心，觀察債務變化。 | 對交易按下「確認」。 | 筆電 A 交易狀態更新為「已確認」，債務金額同步變更。 |
| 5 | 執行「債務結算 (Settle)」。 | 查看結算方案。 | 雙方顯示的結算方案一致。 |

---

## 故障排除

| 問題 | 可能原因 | 解決方法 |
| :--- | :--- | :--- |
| 客戶端顯示 `Mode: Local` | 連線失敗或網址錯誤 | 確認 IP 位址正確，改用 `http://` 而非 `https://` |
| `ping` 不通 | 防火牆或 AP 隔離 | 暫時關閉筆電 A 的 Windows 防火牆 |
| Port 8000 被占用 | 舊的 Python 程序殘留 | 執行 `Get-Process python \| Stop-Process -Force` |
| 模組找不到 | 缺少相依套件 | 執行 `pip install -r requirements.txt` |
