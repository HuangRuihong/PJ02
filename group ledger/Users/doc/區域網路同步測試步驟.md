# 區域網路 (LAN) 資料同步測試步驟 (v2.0)

本文件描述 **Group Ledger (分帳系統)** 在同一區域網路 (LAN) 下，兩台電腦進行即時資料同步的測試流程。

---

## 角色分工

| 角色 | 執行檔 | 說明 |
| :--- | :--- | :--- |
| **電腦 A（伺服器）** | `server\run_online_A.bat` | 啟動 API 伺服器，負責儲存與處理所有資料 |
| **電腦 B（客戶端）** | `Users\run_online_B.bat` | 連線至伺服器，以網路模式啟動 GUI |

---

## 步驟一：環境配置（選用）

現在您可以透過 `.env` 檔案預設連線資訊，避免每次啟動都要輸入網址：
1.  在 `Users/` 目錄下建立 `.env` 檔案。
2.  填入伺服器 IP（例如）：`API_BASE_URL=http://192.168.1.100:8000`。

---

## 步驟二：啟動伺服器（電腦 A）

1. 雙擊執行 `server\run_online_A.bat`。
2. 畫面會自動顯示本機 IP 位址，例如：
   ```
   [Info] Server IP Address : 192.168.1.100
   [Info] Client should connect to : http://192.168.1.100:8000
   ```
3. 看到 `Uvicorn running on http://0.0.0.0:8000` 代表啟動成功，**保持此視窗開啟**。

---

## 步驟三：啟動客戶端（電腦 B）

1. 確保電腦 B 與電腦 A 連接在 **同一個 Wi-Fi 或區域網路**。
2. 雙擊執行 `Users\run_online_B.bat`。
3. 若未設定 `.env`，請依提示輸入電腦 A 的連線網址（例如 `http://192.168.1.100:8000`）。
4. 程式啟動後，觀察側邊欄左下角的 **同步狀態 (Sync Status)**：
   - 顯示「最後同步: HH:MM:SS」：連線成功。
   - 顯示「連線異常」或「同步失敗」：連線至伺服器失敗。

---

## 步驟四：演示情境與功能驗證

### 🎭 演示案例：Alice 與 Bob 的晚餐分帳
| 步驟 | 電腦 A (Alice) 操作 | 電腦 B (Bob) 觀察 | 預期結果 |
| :--: | :--- | :--- | :--- |
| **1. 建立** | 進入「群組中心」，建立新群組「美食愛好者」，將 6 位數邀請碼告知夥伴。 | 進入「群組中心」，點擊「加入群組」並輸入邀請碼。 | 雙方均能看到成員清單更新。 |
| **2. 提案** | 點擊「新增交易」，金額 $1000，成員選 Alice 與 Bob，描述為「麻辣火鍋」。 | 觀察首頁「活動列表」。 | 列表自動出現「同步中...」動畫，隨後顯示該筆交易。 |
| **3. 確認** | 觀察交易狀態。 | 點擊該筆交易旁的「確認」按鈕。 | 電腦 A 的交易狀態立即從「待確認」變更為「已確認」。 |
| **4. 結算** | 點擊「債務結算」，選擇「智慧抵銷 (Simplified)」模式並執行。 | 觀察狀態列與 UI。 | 狀態顯示「結算中...」並鎖定 UI，完成後雙方債務餘額同步歸零。 |

---

## 步驟五：演示進階技巧 (Pro Tips)

1.  **即時性展示**：在進行「確認」操作時，讓兩台電腦螢幕並排，展示狀態幾乎在 1 秒內同步跳動。
2.  **斷線恢復展示**：演示過程中暫時拔掉網路線或關閉伺服器，展示客戶端顯示「連線異常」警告，網路恢復後自動重連，展現 **魯棒性 (Robustness)**。
3.  **UI 平滑感**：指出資料載入時的「同步中...」微動畫，強調系統的精緻度。

## 故障排除

| 問題 | 可能原因 | 解決方法 |
| :--- | :--- | :--- |
| 顯示 `連線異常` | 網址錯誤或防火牆阻擋 | 檢查 IP 是否變動；暫時關閉電腦 A 防火牆。 |
| 顯示 `ModuleNotFoundError` | 缺少 `python-dotenv` | 執行 `pip install python-dotenv`。 |
| 點擊按鈕無反應 | 伺服器已斷線 | 檢查伺服器視窗是否有錯誤訊息，並確認網路是否連通。 |
| Port 8000 被占用 | 舊程序殘留 | 在 PowerShell 執行 `Get-Process python \| Stop-Process -Force` |
| 模組找不到 | 缺少相依套件 | 執行 `pip install -r requirements.txt` |
