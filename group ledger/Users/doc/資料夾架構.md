## 模組權責說明

### 1. 業務邏輯與智慧層 (Intelligence & Services)
- **`intelligence/`**: 存放系統的大腦。`debt_system.py` 負責處理交易狀態的轉換（PENDING -> CONFIRMED -> SETTLED）以及全域的債務計算。
- **`shared/models.py`**: 定義了整個系統通用的資料格式，確保前後端與資料庫之間的一致性。

### 2. 使用者介面 (Frontend UI)
- **`Users/main_gui.py`**: 應用的主骨幹，負責管理側邊欄導覽與模組分頁的切換。
- **`shared/dialogs.py`**: 封裝了所有彈窗互動，保持介面代碼的簡潔。

### 3. 資料持久化 (Data Layer)
- **`shared/base_service.py`**: 所有 Service 類別的父類別，處理資料庫連線的建立與關閉。
- **`shared/data/`**, **`data/`**: 存放所有持久化數據，包含本地資料庫檔案。

### 4. 自動化工具 (Automation Tools)
- **`tools/`**: 提供開發者與使用者便利的維護工具，包含版本控制上傳與資料庫一鍵更新。
- **`*.bat` (批次檔)**: 提供使用者快速啟動程式、更新資料庫及部署服務的捷徑。

### 5. 伺服器端 (Server)
- **`server/`**: 負責處理多台客戶端的連線請求與資料同步的後端 API 服務。

---

## 開發規範建議
- **引用途徑**: 所有模組引用應基於各子系統 (如 `Users`) 根目錄出發（例：`from shared.models import ...`）。
- **資源存放**: 所有的資料庫變動必須在 `Users/doc/migrations.sql` 同步紀錄。

### 模組化設計理念 (Modularization)

本專案採用**高度解耦**的模組化設計，主要體現在以下三個面向：
1.  **介面與邏輯分離**：UI 介面（如 `main_gui.py`）與業務邏輯（如 `service.py`）完全拆分，確保修改介面時不會影響運算邏輯。
2.  **功能域切分**：將系統切分為「個人帳務」與「群組協作」兩大獨立領域，方便多人同時開發與功能擴展。
3.  **抽象網路層**：利用「門面模式 (Facade Pattern)」封裝網路通訊，讓系統能流暢切換單機與網路連線模式。
4.  **客戶端與伺服器分離**：將客戶端 (`Users/`) 與伺服器端 (`server/`) 物理拆分，架構更加清晰，方便獨立部署。
---
### 核心模組 (Core Modules) 

1.  **啟動與入口 (`run_app.py`, `main_gui.py`)**：
    *   負責系統的初始化與主介面佈局。
    *   亮點：展示各功能模組如何像積木一樣被組裝在一起。
2.  **網路門面模組 (`intelligence/network_facade.py`)**：
    *   這是系統的「大腦」，負責決定資料該儲存在本地還是同步到雲端。
    *   亮點：體現了軟體架構中的抽象化能力。
3.  **群組服務模組 (`groups/group_service.py`)**：
    *   處理群組內部的交易邏輯、成員權限與分帳運算。
    *   亮點：這是本作品最具挑戰性的業務核心。
4.  **後端 API 服務 (`server/api_server.py`)**：
    *   基於 FastAPI 架構，處理多台客戶端的連線請求與資料同步。
    *   亮點：展示了系統具備處理「多人同時在線」的能力。
5.  **資料模型模組 (`shared/models.py`)**：
    *   定義了所有帳務物件的結構，是全系統資料交換的共同語言。
---
### 副核心模組 (Secondary Core Modules) 

1.  **資料庫維護工具 (`tools/db_update.py`)**：
    *   處理資料庫結構的自動升級與修補。
    *   重要性：確保程式碼更新後，舊的使用者資料不會損壞。
2.  **權限與認證模組 (`auth/`)**：
    *   處理使用者登入、註冊與連線安全。
    *   重要性：確保群組私密帳務不會被未授權的人存取。
3.  **共享組件模組 (`shared/dialogs.py`)**：
    *   封裝了所有重複使用的提示框與對話視窗。
    *   重要性：體現了程式碼重用 (Code Reuse) 與 UI 風格的一致性。
4.  **自動化腳本系統 (`.bat` 檔案系列)**：
    *   包含 `run_online_A.bat`（網路模式啟動客戶端）、`run_online_B.bat` (啟動伺服器) 與 `start_tunnel_ssh.bat`（內網穿透）。
    *   重要性：簡化了部署流程，讓非技術人員也能一鍵執行作品。
5.  **技術文檔與日誌 (`doc/`, `.gitignore`)**：
    *   紀錄 SQL 變更語句與管理版本控制忽略清單，並有完整的副函式與架構說明。
    *   重要性：展現出這是一個具備正規開發規範的成熟專案。
